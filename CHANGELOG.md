# 更新日志

## [v1.0.5] - 2024-03-31

### 新增功能 ✨

#### 文件操作命令完整实现
- ✅ `rename` - 重命名文件或文件夹
- ✅ `rm` - 删除文件或文件夹
- ✅ `mv` - 移动文件或文件夹
- ✅ `cp` - 复制文件或文件夹
- ✅ `download` - 获取文件下载链接

#### 功能详情

**重命名命令**
```bash
cloud189 rename <文件路径> <新名称>
cloud189 rename /文档/test.txt new_name.txt
cloud189 rename /文档/文件夹 --family  # 家庭云
```

**删除命令**
```bash
cloud189 rm <文件路径>
cloud189 rm /文档/test.txt
cloud189 rm /文档/test.txt --force      # 强制删除，不确认
cloud189 rm /文档/test.txt --permanent  # 永久删除（清空回收站）
cloud189 rm /文档/test.txt --family     # 家庭云
```

**移动命令**
```bash
cloud189 mv <源路径> <目标路径>
cloud189 mv /文档/test.txt /备份/
cloud189 mv test.txt /备份/new_name.txt  # 移动并重命名
cloud189 mv /文档/test.txt /备份/ --family
```

**复制命令**
```bash
cloud189 cp <源路径> <目标路径>
cloud189 cp /文档/test.txt /备份/
cloud189 cp test.txt /备份/copy.txt
cloud189 cp /文档/test.txt /备份/ --family
```

**下载命令**
```bash
cloud189 download <云端路径> <本地路径>
cloud189 download /文档/test.txt ./
cloud189 download /视频/movie.mp4 ./downloads/
cloud189 download /文档/test.txt ./ --family
```

#### 技术实现
- 所有命令支持个人云和家庭云
- 支持绝对路径和相对路径
- 支持文件和文件夹操作
- 友好的错误提示

#### 里程碑
- 🎉 文件操作核心功能全部完成
- 📊 项目完成率达到 51%

## [v1.0.2] - 2024-03-31

### 修复 🐛

#### JSON解析错误修复
- **问题**: 执行 `cloud189 ls /` 等命令时报错 "unexpected end of JSON input"
- **原因**: API响应数据未正确解析，导致所有返回值为空
- **影响**: 所有文件操作、家庭云操作、批量任务功能均无法正常工作

#### 修复详情
修复了以下函数中的JSON解析问题：
- `ListFiles()` - 文件列表查询
- `CreateFolder()` - 创建文件夹
- `GetDownloadURL()` - 获取下载链接
- `GetFamilyList()` - 获取家庭云列表
- `GetCapacityInfo()` - 获取容量信息
- `CreateBatchTask()` - 创建批量任务
- `CheckBatchTask()` - 检查任务状态
- `CreateBatchTaskWithOther()` - 创建批量任务（带额外参数）

#### 技术细节
```go
// 修复前（错误）
var resp SomeType
_, err := c.Get(url, callback, isFamily)  // 返回值被忽略
return &resp, nil  // resp 是零值

// 修复后（正确）
var resp SomeType
data, err := c.Get(url, callback, isFamily)  // 保存返回值
if err := json.Unmarshal(data, &resp); err != nil {  // 解析JSON
    return nil, err
}
return &resp, nil
```

## [v1.0.1] - 2024-03-31

### 修复 🐛

#### 二维码识别问题修复
- **问题**: 生成的ASCII二维码无法扫描识别，定位角缺失
- **原因**: 使用PNG图像解码方式导致二维码结构失真
- **解决方案**: 直接使用 `qr.Bitmap()` 获取二维码位图矩阵

#### 改进细节
1. ✅ 使用 `qrcode.High` 容错级别替代 `Medium`，提高识别率
2. ✅ 直接获取位图矩阵，避免图像解码带来的失真
3. ✅ 添加标准白色边框（border=2），符合二维码规范
4. ✅ 使用 `██` 和 `  ` 字符，确保模块显示正确
5. ✅ 定位角现在清晰可见（三个7x7的定位图案）

### 技术改进

#### 修改文件
- `internal/api/auth.go` - 重写 `printQRCode()` 函数

#### 代码对比
```go
// 改进前：图像解码方式（有失真）
qr.PNG(256)
image.Decode(pngData)  // 这里会导致定位角失真

// 改进后：直接位图方式（无失真）
bitmap := qr.Bitmap()  // 直接获取位图矩阵
for y := 0; y < size; y++ {
    for x := 0; x < size; x++ {
        if bitmap[y][x] {  // 直接使用布尔值
            line += "██"
        } else {
            line += "  "
        }
    }
}
```

#### 容错级别对比
| 级别 | 数据恢复能力 | 适用场景 |
|------|--------------|----------|
| Low | 7% | 高质量打印 |
| Medium | 15% | 普通打印 |
| Quartile | 25% | 一般损坏 |
| **High** | **30%** | **终端ASCII显示（推荐）** |
