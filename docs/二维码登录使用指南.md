## 快速开始

### 1. 启动二维码登录

```bash
.\cloud189.exe login --qr
```

## 二维码结构说明

### 定位角（重要！）

二维码有三个定位角，位于：
- **左上角**
- **右上角**  
- **左下角**

每个定位角是一个 **7x7** 的方块，结构如下：

```
███████████  ← 外圈黑色（7个模块）
█         █
█  █████  █  ← 中间白色（5个模块）
█  █████  █
█  █████  █  ← 中心黑色（3个模块）
█  █████  █
█         █
███████████
```

### 为什么之前无法识别？

❌ **旧实现问题**：
- 使用PNG图像解码
- 图像缩放导致定位角失真
- 字符选择不当

✅ **新实现改进**：
- 直接使用位图矩阵
- 无图像缩放
- 使用 `██` 双字符确保比例正确

## 常见问题

### Q1: 二维码显示不完整怎么办？

**A**: 
1. 确保终端窗口足够大
2. 调整终端字体大小（建议：Consolas 或等宽字体）
3. 不要缩放终端窗口

### Q2: 扫码后提示"二维码已过期"？

**A**: 
二维码有效期约 **5分钟**，请重新运行登录命令：
```bash
.\cloud189.exe login --qr
```

### Q3: 无法识别二维码？

**A**: 
1. ✅ 确保使用最新版本（v1.0.1+）
2. ✅ 检查定位角是否清晰可见
3. ✅ 调整手机与屏幕的距离和角度
4. ✅ 提高终端亮度

### Q4: 扫码后一直等待？

**A**: 
程序每 **5秒** 检查一次状态，最长等待 **5分钟**

## 备用方案

### 方案1: 浏览器打开

如果终端二维码无法识别，可以：

1. 复制终端显示的链接
2. 在浏览器中打开
3. 使用APP扫描浏览器中的二维码

链接格式：
```
https://m.cloud.189.cn/zhuanti/qrLogin/qrCode/index.html?uuid=...
```

### 方案2: 密码登录

如果二维码登录始终有问题，可以使用密码登录：

```bash
.\cloud189.exe login -u 用户名 -p 密码
```

## 技术细节

### 容错级别

我们使用 **High** 级别容错，可以恢复 **30%** 的数据：

| 级别 | 恢复能力 | 适用场景 |
|------|----------|----------|
| Low | 7% | 高质量打印 |
| Medium | 15% | 普通打印 |
| Quartile | 25% | 轻微损坏 |
| **High** | **30%** | **终端ASCII显示** |

### 为什么选择 High 级别？

终端ASCII显示可能出现：
- 字符显示不完整
- 行间距问题
- 颜色对比度不足

High 级别容错可以最大程度保证识别成功率。

## 最佳实践

### 推荐的终端设置

```
字体: Consolas / Monaco / Courier New
字号: 14-16pt
行距: 1.0-1.2倍
终端宽度: 至少80列
```

1. ✅ 保持手机与屏幕 **10-20cm** 距离
2. ✅ 确保光线充足，避免反光
3. ✅ 手机保持 **垂直** 于屏幕
4. ✅ 等待相机 **自动对焦**
