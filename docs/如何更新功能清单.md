# 如何更新功能清单

当完成一个新功能时，请按以下步骤更新文档：

## 1. 更新 `功能清单.md`

### 步骤：

1. **找到对应功能项**
   - 在"已实现功能"或"待实现功能"中找到对应条目

2. **更新状态**
   ```markdown
   # 从待实现移到已实现
   | ⬜ 删除文件 | `rm <路径>` | - | 待实现 |
   # 改为
   | ✅ 删除文件 | `rm <路径>` | v1.0.5 | 已实现 |
   ```

3. **更新进度概览**
   ```markdown
   | 文件操作 | 3 | 0 | 7 | 30% |
   # 更新数字
   | 文件操作 | 4 | 0 | 6 | 40% |
   ```

4. **更新进度图**
   ```markdown
   文件操作  ████████░░░░░░░░░░░░  30%
   # 增加进度
   文件操作  ██████████░░░░░░░░░░  40%
   ```

5. **添加更新记录**
   ```markdown
   ### v1.0.5 (2024-03-31)
   - ✅ 新增：`rm` 删除文件命令
   - ✅ 新增：`mv` 移动文件命令
   ```

## 2. 更新 `README.md`

更新开发计划部分：

```markdown
### 当前进度
- ✅ 已完成：16项功能
- ⬜ 待实现：25项功能
- 📊 完成率：39%
```

## 3. 更新 `CHANGELOG.md`

添加新版本记录：

```markdown
## [v1.0.5] - 2024-03-31

### 新增功能 ✨

#### 文件删除命令
- 支持删除文件和文件夹
- 支持强制删除（--force）
- 支持永久删除（--permanent）

#### 文件移动命令
- 支持移动文件和文件夹
- 支持重命名移动
```

## 4. 更新 `PROJECT_SUMMARY.md`

移动已完成的功能：

```markdown
### ✅ 文件操作
- [x] 文件列表
- [x] 创建文件夹
- [x] 删除文件      # 新增
- [x] 移动文件      # 新增
```

---

## 快速检查清单

完成功能后，确保更新了以下文件：

- [ ] `功能清单.md` - 主要清单文件
- [ ] `README.md` - 项目说明
- [ ] `CHANGELOG.md` - 更新日志
- [ ] `PROJECT_SUMMARY.md` - 项目总结

## 功能状态标记说明

| 标记 | 含义 |
|------|------|
| ✅ | 已实现并测试通过 |
| ⬜ | 待实现 |
| 🔄 | 开发中 |
| ⚠️ | 部分实现 |
| ❌ | 已弃用 |

---

## 优先级说明

| 级别 | 说明 | 示例 |
|------|------|------|
| P0 | 核心功能，必须实现 | 删除、移动、下载 |
| P1 | 重要功能，建议实现 | 上传、进度显示 |
| P2 | 增强功能，可选实现 | 秒传、断点续传 |
| P3 | 高级功能，未来规划 | 搜索、分享、同步 |

---

## 版本号规范

遵循语义化版本规范：

- **主版本号（Major）**: 不兼容的API更改
- **次版本号（Minor）**: 向下兼容的功能新增
- **修订号（Patch）**: 向下兼容的问题修正

示例：
- v1.0.0 → v1.0.1: Bug修复
- v1.0.1 → v1.1.0: 新增功能
- v1.1.0 → v2.0.0: 重大更新
