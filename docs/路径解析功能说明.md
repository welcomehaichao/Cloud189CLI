# 路径解析功能说明

## 概述

从 v1.0.4 版本开始，Cloud189CLI 支持完整的路径解析功能，可以使用绝对路径和相对路径访问文件和文件夹。

## 支持的路径格式

### 1. 绝对路径

以 `/` 开头的路径，从根目录开始解析。

```bash
# 访问根目录
.\cloud189.exe ls /

# 访问一级文件夹
.\cloud189.exe ls /文档

# 访问多级文件夹
.\cloud189.exe ls /文档/工作/项目
```

### 2. 相对路径

不以 `/` 开头的路径，从根目录开始解析（目前不支持相对于当前工作目录）。

```bash
# 相当于 /文档
.\cloud189.exe ls 文档

# 相当于 /文档/工作
.\cloud189.exe ls 文档/工作
```

## 路径解析规则

### 1. 根目录

- 个人云根目录ID: `-11`
- 家庭云根目录ID: 空字符串

### 2. 路径分隔符

使用 `/` 作为路径分隔符（与Unix/Linux相同）。

### 3. 特殊目录

- `/` 或空字符串: 根目录
- `.`: 当前目录（保留，暂未实现）
- `..`: 父目录（保留，暂未实现）

## 功能支持

### ls 命令

列出指定路径下的文件和文件夹：

```bash
# 列出根目录
.\cloud189.exe ls /

# 列出指定文件夹
.\cloud189.exe ls /文档

# 支持表格格式
.\cloud189.exe ls /文档 -o table
```

### mkdir 命令

在指定路径创建文件夹：

```bash
# 在根目录创建
.\cloud189.exe mkdir 新文件夹

# 在指定路径创建
.\cloud189.exe mkdir /文档/新文件夹

# 支持多级创建
.\cloud189.exe mkdir /文档/工作/新项目
```

**注意**：如果父路径不存在，命令会失败。需要先创建父文件夹。

## 错误处理

### 路径不存在

```bash
.\cloud189.exe ls /不存在的路径

错误: failed to resolve path '/不存在的路径': folder '不存在的路径' not found at path '/不存在的路径'
```

### 权限不足

```bash
.\cloud189.exe ls /其他用户的文件夹

错误: failed to list files: API error: Permission denied
```

## 性能优化

### 路径缓存

路径解析器会缓存已解析的路径，避免重复查询：

```bash
# 第一次查询，需要遍历路径
.\cloud189.exe ls /文档/工作

# 第二次查询，使用缓存
.\cloud189.exe ls /文档/工作
```

### 分页支持

路径解析使用分页查询，支持大量文件：

```bash
.\cloud189.exe ls /大文件夹 --page 1 --page-size 1000
```

## 最佳实践

### 1. 使用绝对路径

推荐使用绝对路径，避免歧义：

```bash
# 推荐
.\cloud189.exe ls /文档/工作

# 不推荐（虽然支持）
.\cloud189.exe ls 文档/工作
```

### 2. 路径格式化

路径末尾的 `/` 会被自动处理：

```bash
# 这两个命令等效
.\cloud189.exe ls /文档/
.\cloud189.exe ls /文档
```

### 3. 创建多级文件夹

逐级创建，确保父路径存在：

```bash
# 先创建父文件夹
.\cloud189.exe mkdir /文档
.\cloud189.exe mkdir /文档/工作
.\cloud189.exe mkdir /文档/工作/项目
```

## 实现细节

### PathResolver 类

```go
type PathResolver struct {
    client     *Client
    cache      map[string]string  // 路径缓存
    cacheMutex sync.RWMutex
    isFamily   bool
}

// 解析路径到文件夹ID
func (pr *PathResolver) ResolvePath(path string) (string, error)

// 解析路径，不存在则创建
func (pr *PathResolver) ResolvePathWithCreate(path string, create bool) (string, error)
```

### 解析流程

1. 检查缓存
2. 分割路径
3. 从根目录开始逐级查找
4. 验证每个路径段是否存在
5. 缓存解析结果

## 版本历史

- **v1.0.4**: 初始实现，支持绝对路径和相对路径
- **计划**: 支持相对当前工作目录、父目录导航（..）

## 常见问题

### Q: 为什么相对路径从根目录开始？

A: 目前CLI没有维护"当前工作目录"的概念，所有相对路径都从根目录解析。未来版本会支持类似 `cd` 的命令。

### Q: 如何处理空格？

A: 路径中的空格不需要特殊处理，直接使用：

```bash
.\cloud189.exe ls /我的 文档
```

### Q: 支持通配符吗？

A: 暂不支持通配符。计划在未来版本中添加。

### Q: 路径长度有限制吗？

A: 没有明确的路径深度限制，但每级解析都需要API调用，深层路径会影响性能。
